## Завдання 1. Базові команди 

### 1. Показати вміст домашнього каталогу

```bash
ls ~
```

**Результат:** відображається список файлів і папок у домашньому каталозі (`/home/username`).  
Якщо каталог порожній — виводу немає.

---

### 2. Перейти у каталог Downloads і показати його вміст

```bash
cd ~/Downloads
ls
```

**Результат:** термінал переходить у папку `Downloads`, команда `ls` виводить її вміст.  
Якщо папка порожня — виводу немає.

---

### 3. Створити порожній файл (без `touch`)

```bash
echo "" > myfile.txt
```

**Пояснення:** команда `echo ""` виводить порожній рядок, а оператор `>` перенаправляє вивід у файл `myfile.txt`. Файл створюється без використання `touch`.

---

### 4. Переглянути вміст файлу

```bash
cat myfile.txt
```

**Результат:** оскільки файл порожній (або містить один порожній рядок), виводиться або нічого, або один порожній рядок.

---

### 5. Перейти у домашній каталог абсолютним шляхом

```bash
cd /home/alexandramohylda
```

**Пояснення:** абсолютний шлях починається з кореня файлової системи `/` і вказує повний маршрут до каталогу.

---

### 6. Перейти у домашній каталог відносним шляхом

```bash
cd ~
```

або (якщо ти знаходишся на рівень вище):

```bash
cd ../alexandramohylda
```

**Пояснення:** `~` — це скорочення для домашнього каталогу поточного користувача. Відносний шлях вказується відносно поточної позиції в файловій системі, а не від кореня.

## Завдання 2. Робота з документацією

### Виконані команди

```bash
man ls
help cd
man cat
man man
cp --help
mv --help
```

---

### Питання 1. Який ключ `ls` показує приховані файли?

```bash
ls -a
```

**Відповідь:** ключ `-a` (від англ. *all*) показує всі файли, включаючи приховані — ті, чиї імена починаються з крапки (наприклад, `.bashrc`, `.profile`).  
Також існує варіант `-A` — показує приховані файли, але **без** `.` і `..` (поточний і батьківський каталог).

---

### Питання 2. Який ключ у `cat` нумерує рядки?

```bash
cat -n filename.txt
```

**Відповідь:** ключ `-n` (від англ. *number*) нумерує всі рядки файлу при виводі.  
Існує також `-b` — нумерує тільки непорожні рядки.

---

### Питання 3. Чим відрізняються `man` і `--help`?

| Характеристика | `man` | `--help` |
|---|---|---|
| Повнота | Повна документація з усіма секціями | Коротка довідка з основними опціями |
| Спосіб виводу | Відкриває інтерактивний переглядач (`less`) | Виводить текст одразу у термінал |
| Навігація | Можна гортати, шукати (`/`), виходити (`q`) | Прокрутка лише колесом миші або скролом |
| Доступність | Потребує встановленого пакету `man-db` | Вбудована у більшість команд |
| Приклад | `man ls` | `ls --help` |

**Висновок:** `man` підходить для детального вивчення команди, а `--help` — для швидкого перегляду доступних ключів.

## Завдання 3. Міні-сценарій

**Мета:** продемонструвати роботу з файловою системою — переміщення, створення файлів, перегляд вмісту та документації.

### Сценарій: робота з тимчасовою папкою

```bash
# Крок 1: Перейти в каталог /tmp
cd /tmp

# Крок 2: Створити файл (без touch)
echo "Це мій тестовий файл" > notes.txt

# Крок 3: Переглянути список файлів у поточному каталозі
ls -la

# Крок 4: Переглянути вміст створеного файлу
cat notes.txt

# Крок 5: Перейти в інше місце — домашній каталог
cd ~

# Крок 6: Повернутись назад у /tmp
cd /tmp

# Крок 7: Ще раз переглянути список файлів (переконатись, що файл на місці)
ls

# Крок 8: Переглянути документацію команди ls
man ls
```

### Пояснення сценарію

| Крок | Команда | Що робить |
|---|---|---|
| 1 | `cd /tmp` | Переходимо у тимчасовий каталог (абсолютний шлях) |
| 2 | `echo "..." > notes.txt` | Створюємо файл і записуємо в нього текст |
| 3 | `ls -la` | Переглядаємо список файлів з деталями (права, розмір, дата) |
| 4 | `cat notes.txt` | Читаємо вміст файлу |
| 5 | `cd ~` | Переходимо у домашній каталог |
| 6 | `cd /tmp` | Повертаємось назад у `/tmp` |
| 7 | `ls` | Переконуємось, що файл `notes.txt` збережений |
| 8 | `man ls` | Переглядаємо документацію команди `ls` |
